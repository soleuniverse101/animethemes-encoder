# Changesets

This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help version and publish code. You can
find the full documentation for it [in this repository](https://github.com/changesets/changesets).

You can also check out a quick list of common questions to get you started engaging with this project in
[our documentation](https://github.com/changesets/changesets/blob/main/docs/common-questions.md).

## Workflow details

Currently, this project is in early development phase, so versioning and releasing are experimental and not quite defined yet, but the most important rule is just not to bump the major version. So we stay in `v0.x.x` and we can basically treat it like `v0.MAJOR/MINOR.MINOR/PATCH`. Simple patches that are not significant (e.g. refactoring/fixes) are less likely to provoke a version bump while relatively important new features should probably require a major (`v0.x`) bump.
